## 1. Концепция и позиционирование

### 1.1 Elevator Pitch

«Репа» — мобильное Flutter-приложение (iOS + Android), в котором пользователи состоят в закрытых группах (класс, компания, чат) и еженедельно анонимно голосуют за участников по смешным и провокационным вопросам. В конце недели каждый получает персональную карточку репутации. Можно оставить себе или поделиться — решает сам игрок. Группа может быть связана с Telegram-каналом: для быстрого перехода в чат и шаринга результатов.

### 1.2 Целевая аудитория

| Параметр | Описание |
|---|---|
| Возраст | 14–22 года (ядро: 15–19, школьники и студенты) |
| Устройства | iPhone iOS 16+ и Android 10+, приоритет бюджетный и средний сегмент |
| Мотивация | Хочу знать, что обо мне думают, не спрашивая напрямую |
| Страх | Быть оценённым несправедливо или не вписаться в группу |
| Среда | Telegram-чаты класса/группы — основной канал коммуникации |
| Паттерн | Короткие сессии 2–5 мин, пики пятница вечер / воскресенье |

### 1.3 Уникальное позиционирование

> **Главная идея:** это не викторина — это зеркало. Ты узнаёшь, как тебя видят люди, которые тебя знают. Анонимность снижает порог честности. Красивая карточка делает результат артефактом для шеринга. Ачивки и статистика формируют «легенду» каждого игрока в группе.

### 1.4 Ключевые метрики успеха (MVP)

| Метрика | Цель (3 мес.) | Пояснение |
|---|---|---|
| DAU/MAU | > 35% | Еженедельный цикл удерживает |
| Retention Day 7 | > 40% | Выше среднего по casual (25%) |
| Retention Day 30 | > 20% | Социальный крючок работает |
| Organic install share | > 60% | Виральность через шеринг карточек |
| Группы на юзера | ≥ 2 | Подтверждение многогрупповой модели |
| ARPU (платящие) | > 180 ₽/мес | Монетизация через детекторы |


### 1.5 Бренд и голос

| Параметр | Значение |
|---|---|
| Название | Репа |
| Произношение | Как сленговое «репутация» — коротко, своё |
| Иконка | 🍆 баклажан (он же «репа» в молодёжном сленге) — готовый мем |
| Слоган | «Чё там в Репе?» |
| Тон пушей | Разговорный, как от друга: «Глянь что о тебе думают», «Твоя репа готова» |
| Глагол | «Порепиться», «репануть», «скинь репу» — название живёт в речи |
| Запрещено | Официоз, корпоративный тон, слово «репутация» в коммуникациях |

## 2. Core Loop и игровые механики

### 2.1 Недельный цикл (Game Loop)

| День | Событие | Действие игрока |
|---|---|---|
| ПН–СР (Сбор) | Открыт новый Сезон. Банк вопросов активен. | Голосуешь анонимно за других участников |
| ЧТ (Напоминание) | Пуш: «Ещё не проголосовал? Осталось 36 часов» | Догоняющие открывают приложение |
| ПТ 20:00 (Reveal) | REVEAL — карточки репутации разблокированы | Смотришь карточку, решаешь — шарить или нет |
| ПТ–ВС (Обсуждение) | Активность в чатах, шеринг, детекторы | Покупаешь детектор, смотришь кто голосовал |
| ВС 23:59 (Сброс) | Сезон закрывается, обновляется статистика и ачивки | Следующий сезон стартует в понедельник |

### 2.2 Механика голосования

**Как работает вопрос:**
Каждый вопрос — атрибут, который нужно присвоить одному участнику группы. Вопросы формулируются в третьем лице, нейтрально-юмористически.

> **Пример:** «Кто первым побежит при пожаре?» → игрок выбирает одного участника из списка. Каждый участник видит только свой % по итогам, не то, кто конкретно его выбрал.

**Правила голосования:**
- Один вопрос — один голос на одного участника
- Нельзя голосовать за себя
- В сессии голосования: 5–10 вопросов подряд (зависит от размера группы)
- После завершения голосования — заблокировано до следующего сезона
- Minimum quorum для Reveal: ≥ 50% участников группы проголосовало
- Пользователь не может изменить голос после подтверждения

### 2.3 Банк вопросов

**Готовые категории:**
- 🔥 Горячие — провокационные, но безопасные
- 😂 Смешные — юмор без обид
- 🤫 Секреты — что скрывает человек
- 💪 Способности — кто на что способен
- ❤️ Романтика — для взрослых групп (18+)
- 🎓 Учёба — для школьных чатов

**Пользовательские вопросы:**
- Создатель группы и любой участник могут добавить свой вопрос
- Новый вопрос проходит автомодерацию (AI, Anthropic API)
- При нарушении правил — отклоняется с пояснением
- Принятый вопрос входит в ротацию группы
- Автор вопроса не раскрывается

### 2.4 Карточка репутации (Reveal Screen)

Персональный итоговый экран — главный артефакт игры. Формируется в момент Reveal в пятницу в 20:00.

| Элемент карточки | Содержание |
|---|---|
| Аватар + ник | Имя и фото профиля игрока |
| Топ-3 атрибута | Три самых часто присвоенных атрибута с % (напр. «Убежит при пожаре 67%») |
| Скрытые атрибуты | Остальные атрибуты — заблюрены, открываются за кристаллы |
| Репутационный титул | Автогенерация по топ-атрибутам (напр. «Хранитель Тайн») |
| Недельный тренд | Сравнение с прошлым сезоном: атрибуты выросли / упали |
| Ачивки сезона | Новые ачивки, полученные по итогам сезона |
| Кнопка шеринга | PNG для Telegram / Stories (игрок выбирает сам) |

## 3. Пользовательские роли и сценарии

### 3.1 Роли

| Роль | Описание | Привилегии |
|---|---|---|
| Участник | Обычный член группы | Голосует, видит свою карточку, покупает детекторы |
| Администратор | Создатель группы (стартер) | Только: задаёт имя группы, привязывает Telegram-канал, добавляет вопросы |

> **Принципиально:** администратор НЕ видит чужие ответы, НЕ может исключать участников, НЕ получает бонусов. Равенство — ключевое для честного восприятия игры.

### 3.2 Ключевые пользовательские сценарии (User Stories)

**US-01 · Создание группы**
1. Пользователь нажимает «Создать группу»
2. Вводит название (max 40 симв.)
3. Выбирает категории вопросов
4. Опционально: привязывает Telegram-канал группы
5. Система генерирует уникальную ссылку-инвайт
6. Пользователь отправляет ссылку в чат/мессенджер
7. Группа активируется при ≥ 3 участниках

**US-02 · Присоединение к группе**
1. Пользователь получает ссылку-инвайт (Telegram, SMS, любой мессенджер)
2. Открывает ссылку → диплинк запускает приложение
3. Если нет аккаунта → быстрая регистрация (Apple/Google или телефон)
4. Подтверждает участие → попадает в группу
5. Один пользователь может состоять в ≤ 10 группах одновременно (MVP)

**US-03 · Голосование**
1. Уведомление: «Сезон начался в группе [Название]»
2. Открывает сессию голосования
3. Видит вопрос + карточки участников
4. Выбирает одного участника → свайп или тап
5. Проходит 5–10 вопросов подряд
6. Может прервать и вернуться позже (прогресс сохраняется)

**US-04 · Reveal Friday**
1. В пятницу в 20:00 → пуш «Твоя карточка готова»
2. Открывает Reveal Screen с анимацией
3. Видит топ-3 атрибута + ачивки сезона, остальное заблюрено
4. Решает: купить детектор / открыть скрытые / поделиться / закрыть
5. Если шарит → выбирает платформу (включая Telegram-канал группы)

**US-05 · Покупка детектора**
1. На Reveal Screen нажимает «Узнать, кто голосовал»
2. Видит цену в кристаллах
3. Если кристаллов нет → попадает в магазин
4. Оплачивает через ЮKassa (внешний биллинг)
5. Детектор активируется: показывает список участников, голосовавших в данном сезоне

> ⚠️ Детектор показывает только «кто участвовал в голосовании», но НЕ «кто как ответил».

**US-06 · Просмотр статистики и ачивок**
1. Открывает профиль участника в группе
2. Видит историю атрибутов по сезонам
3. Видит коллекцию ачивок с датой получения
4. Видит «легенду» — автогенерируемое описание игрока в этой группе

## 5. Статистика и ачивки

### 5.1 Концепция

Каждый участник в каждой группе ведёт накопительную статистику по сезонам. Ачивки — это публичные маркеры «легенды» игрока в конкретной группе. Они отображаются на профиле, на карточке Reveal и в списке участников группы.

> Статистика и ачивки **контекстуальны**: они привязаны к группе, а не к аккаунту глобально. У игрока могут быть разные «легенды» в разных группах.

### 5.2 Статистика участника (по группе)

| Метрика | Описание |
|---|---|
| Сезонов сыграно | Сколько раз участвовал в голосовании |
| Атрибутов получено | Общее количество атрибутов за все сезоны |
| Топ-атрибут всех времён | Самый часто присваиваемый атрибут за историю группы |
| Атрибут этого сезона | Лидирующий атрибут текущего сезона |
| Точность предсказания | % совпадений: за кого голосовал сам vs кто реально «выиграл» вопрос |
| Лучший знаток | За кого из конкретных участников предсказывает точнее всего |
| Стабильность репутации | Насколько атрибуты повторяются от сезона к сезону |

### 5.3 Банк ачивок

#### 🎯 Ачивки угадывания (за точность голосования)

| Ачивка | Условие |
|---|---|
| 🎯 Снайпер | Угадал правильного участника в 80%+ вопросов за сезон |
| 🔮 Оракул | 3 сезона подряд точность угадывания > 70% |
| 💫 Телепат | Угадал всех правильно в одной сессии голосования |
| 🙈 Вслепую | Точность < 20% в сезоне (антиачивка, юмористическая) |
| 🎲 Рандомщик | Каждый раз выбирает разного участника, паттерна нет |

#### 👥 Ачивки знания конкретных людей

| Ачивка | Условие |
|---|---|
| 📖 Знаток [Имя] | Угадывал правильно за конкретного участника 5+ раз подряд |
| 🤝 Лучший друг [Имя] | Взаимное: оба часто угадывают друг друга |
| 🕵️ Детектив | Угадал 3 разных участника в одном сезоне с точностью 100% |
| 💔 Незнакомец | Ни разу не угадал за конкретного участника (после 5+ попыток) |

#### 🏆 Ачивки репутации (за получаемые атрибуты)

| Ачивка | Условие |
|---|---|
| 👑 Легенда группы | Один и тот же атрибут лидирует 5 сезонов подряд |
| 🌪️ Переменчивый | Топ-атрибут меняется каждый сезон (5+ сезонов) |
| 🥇 Монополист | Получает 70%+ голосов по одному вопросу |
| 🎭 Загадка | Атрибуты каждый сезон непредсказуемы, нет паттерна |
| 📈 На подъёме | 3 сезона подряд растёт % по ключевому атрибуту |
| 🌟 Первый | Первым получил данный атрибут в этой группе |

#### ⚡ Ачивки активности

| Ачивка | Условие |
|---|---|
| 🔥 Не пропускаю | Проголосовал в 10 сезонах подряд |
| ⚡ Первый | Первым завершил голосование в сезоне |
| 🐌 Последний | Проголосовал последним (но всё же проголосовал) |
| 🌙 Сова | Голосует в промежутке 23:00–03:00 3+ раза |
| 🧐 Аналитик | Использовал детектор в 5+ сезонах |

#### 🎁 Ачивки шеринга и социальности

| Ачивка | Условие |
|---|---|
| 📣 Медийная личность | Поделился карточкой в 5+ сезонах подряд |
| 🤐 Конспиратор | Ни разу не поделился карточкой (10+ сезонов) |
| 🚀 Вирус | Привёл в группу 3+ новых участников по инвайт-ссылке |

### 5.4 Отображение ачивок

- На карточке Reveal: новые ачивки сезона — с анимацией разблокировки
- В профиле участника в группе: полная коллекция в хронологическом порядке
- В списке участников группы: топ-3 ачивки видны всем как бейджи
- Ачивки НЕ удаляются при выходе из группы — остаются в архиве

### 5.5 «Легенда» участника

Автоматически генерируемый текст-описание на основе топ-атрибутов и ачивок. Отображается на странице профиля в группе.

> **Пример:** *«Серёга — настоящий Снайпер: угадывает всех лучше всех в группе. Главная его репутация — «Убежит при пожаре», и так уже 4 сезона подряд. Детектив и Легенда группы в одном лице.»*

Генерируется по шаблону на клиенте (без AI в MVP) на основе набора ачивок и топ-атрибута.

## 6. Интеграция с Telegram

### 6.1 Принцип: интеграция полностью опциональна

Telegram-интеграция — enhancement, не core. Все функции Репы работают без неё.
Архитектурно: поле `telegram_chat_id` в таблице `groups` — nullable. Весь Telegram-функционал
живёт за одной проверкой `if group.telegram_chat_id is not null`. Если null — ветка не вызывается,
никакой условной логики по всему коду.

**Три режима использования:**

| Режим | Описание | Что нужно |
|---|---|---|
| 1 — Только приложение | Telegram не используется вообще | Ничего |
| 2 — Ссылка на чат | Кнопка «Открыть Telegram-чат» → просто URL | Вставить username чата в настройках группы |
| 3 — Полная интеграция | Бот сидит в чате, постит автоматически | Добавить @repa_bot в чат как администратора |

### 6.2 Режим 2 — ссылка на чат (без бота)

Минимальная интеграция. Администратор вводит username Telegram-чата (`@classname`) в настройках группы.
В приложении появляется кнопка «Открыть чат» — открывает Telegram через `tg://` deeplink с fallback на `https://t.me/`.
Бот не нужен, никаких прав, никакой верификации.

| ID | Требование | Приоритет |
|---|---|---|
| TG-01 | Поле «Username Telegram-чата» в настройках группы (опционально) | P1 |
| TG-02 | Кнопка «Открыть чат» на экране группы при заполненном username | P1 |
| TG-03 | Открытие через `tg://` с fallback на `https://t.me/username` | P1 |

### 6.3 Режим 3 — бот в чате

#### Подключение бота

Администратор группы в Репе инициирует подключение:

1. Нажимает «Подключить Telegram» в настройках группы
2. Приложение генерирует уникальный **connect-код** (например `REPA-X7K2`), хранится на backend с TTL 24 часа
3. Администратор добавляет `@repa_bot` в свой Telegram-чат с правами администратора
4. Пишет в чат: `/connect REPA-X7K2`
5. Бот проверяет код на backend → связывает `chat_id` с группой → подтверждает в чате
6. Connect-код инвалидируется, связка сохраняется в `groups.telegram_chat_id`

> Почему код, а не OAuth: у школьников нет «авторизации через Telegram» — у них есть чат класса.
> Код — самый простой UX для их контекста.

| ID | Требование | Приоритет |
|---|---|---|
| TG-04 | Генерация уникального connect-кода с TTL 24 часа | P1 |
| TG-05 | Команда `/connect CODE` — верификация и привязка chat_id к группе | P1 |
| TG-06 | Бот запрашивает только право `can_post_messages` — не читает сообщения чата | P0 |
| TG-07 | Подтверждение успешной привязки в чате и в приложении | P1 |
| TG-08 | Отвязка Telegram-чата из настроек группы в приложении | P1 |
| TG-09 | При удалении бота из чата — автоматическая отвязка (обработка `my_chat_member` event) | P1 |

#### Что бот умеет

Бот работает в двух направлениях: пишет в чат по расписанию и отвечает на команды.

**Автоматические посты (scheduled):**

| Триггер | Время | Содержание поста |
|---|---|---|
| Старт сезона | ПН 17:00 | «🍆 Новый сезон начался — голосуй в приложении» + кнопка-ссылка |
| Напоминание (низкий кворум) | СР 20:00 при кворуме < 50% | «Reveal под угрозой — не хватает X голосов» + кнопка |
| Тизер Reveal | ПТ 19:00 | «Через час узнаете всё. Или не всё 👀» |
| Reveal-пост | ПТ 20:00 | Агрегированные итоги без деталей + кнопки на карточки участников |
| Ачивка группы | ПТ 20:00 (после Reveal) | «[Имя] получил ачивку Легенда группы 👑» |

**Команды участников:**

| Команда | Кто может | Ответ бота |
|---|---|---|
| `/repa` | Все | Статус текущего сезона: сколько проголосовало, сколько до Reveal |
| `/connect CODE` | Администратор | Привязка группы (одноразово) |
| `/disconnect` | Администратор | Отвязка бота от группы |

> Бот отвечает на команды публично в чате, кроме `/mystats` — его убираем из MVP
> (требует идентификации Telegram user_id с аккаунтом в Репе, это отдельная сложность).

**Ручной шеринг карточки:**

Игрок в приложении нажимает «Поделиться в Telegram-чат группы» →
backend публикует PNG карточки в привязанный чат через Bot API.
Игрок сам решает — шарить или нет. Автошеринга без согласия нет.

| ID | Требование | Приоритет |
|---|---|---|
| TG-10 | Автопост старта сезона в понедельник | P1 |
| TG-11 | Автопост напоминания при низком кворуме | P1 |
| TG-12 | Автопост тизера в пятницу 19:00 | P1 |
| TG-13 | Автопост Reveal-итогов в пятницу 20:00 | P1 |
| TG-14 | Reveal-пост содержит только агрегат и ссылки — без детальных карточек | P0 |
| TG-15 | Кнопка «Поделиться в чат группы» на Reveal Screen — ручной шеринг PNG | P1 |
| TG-16 | Команда `/repa` — статус сезона | P1 |
| TG-17 | Все Telegram-посты содержат deeplink обратно в приложение | P0 |

#### Формат Reveal-поста (Режим 3)

Детали (кто именно получил какой атрибут) — только в приложении.

### 6.4 Что никогда не делает бот

- Не читает сообщения чата (только получает команды через Telegram Bot API)
- Не собирает данные об участниках чата
- Не публикует индивидуальные карточки без явного действия игрока
- Не хранит историю переписки

### 6.5 Технические детали

| Параметр | Значение |
|---|---|
| Bot API | Telegram Bot API (Long Polling в dev, Webhook в prod) |

| Права бота | `can_post_messages` — минимально необходимые |
| Публикация PNG | Загрузка через `sendPhoto` с уже сгенерированным S3-URL |

### 7.1 Модель

> Игра бесплатна. Монетизация — через внутреннюю валюту «кристаллы», которые покупаются через ЮKassa (внешний биллинг, не Apple/Google IAP). Это позволяет избежать 30% комиссии сторов и строить собственную воронку.

| Продвинутый | 299 ₽ | 70 | +15 | 7 детекторов + месяц атрибутов |

### 8.2 Критические архитектурные решения

**Анонимность голосований**

Связь «голосующий → голос» хранится в БД (необходима для детектора), но НИКОГДА не передаётся в клиент в явном виде. Детектор показывает только список участвовавших в голосовании — без атрибуции конкретных ответов. Это security requirement первого приоритета.

**Reveal как scheduled event**

Каждый активный сезон имеет запись в scheduling таблице. Redis job проверяет каждую минуту. В 20:00 пятницы: проверка quorum → вычисление агрегатов → расчёт ачивок → запись результатов → отправка FCM push → разблокировка Reveal Screen → (опционально) публикация в Telegram.

**Расчёт ачивок**

Ачивки рассчитываются на backend в момент завершения сезона. Алгоритм пробегает по таблице votes за текущий и предыдущие сезоны. Результат записывается в таблицу achievements с датой и сезоном. Клиент только отображает.

**Карточка Reveal**

PNG генерируется на backend (Puppeteer или node-canvas) с данными пользователя. Хранится в S3 с TTL 30 дней. URL передаётся в Flutter-приложение. Нативный шерер: `Share.shareXFiles()` из пакета `share_plus`.

**Flutter-специфика**

- Deeplinks: пакет `app_links` для Universal Links (iOS) и App Links (Android)
- Push: `firebase_messaging` — единый для обеих платформ
- Платёжная страница: `url_launcher` → внешний браузер → диплинк возврата
- Telegram открытие: `url_launcher` с `tg://` схемой с fallback на https

### 8.3 Схема данных (основные сущности)

## 9. Безопасность и модерация

### 9.1 Защита анонимности

- Голоса хранятся в зашифрованной таблице с разделением app-user и vote-data
- API никогда не возвращает связку vote_id + voter_id + answer в одном ответе
- Детектор активируется отдельным endpoint, который возвращает только voter list
- Rate limiting на все чувствительные endpoints
- Pen-test перед запуском: проверка невозможности деанонимизации через API

### 9.2 Модерация контента

| Тип контента | Метод | Действие при нарушении |
|---|---|---|
| Пользовательские вопросы | AI (Claude API) — синхронно при создании | Отклонение + объяснение. Повторные нарушения → бан создания вопросов |
| Никнеймы | AI + список стоп-слов | Блокировка регистрации с таким ником |
| Жалоба на вопрос | Ручная проверка (отложенная в MVP) | Удаление вопроса из банка группы |
| Жалоба на пользователя | Ручная проверка | Предупреждение или бан аккаунта |
| Telegram-пост автошеринга | Проверка через тот же AI pipeline | Пост не публикуется, уведомление администратору |

### 9.3 Защита несовершеннолетних

> Аудитория 14+. Обязательный ввод года рождения при регистрации. Для пользователей до 18 лет: категория «Романтика» в банке вопросов недоступна для выбора при создании группы.

## 12. Система удержания (Retention System)

### 12.1 Стратегия

Базовый loop даёт 2 естественных захода в неделю: проголосовал + посмотрел результаты.
Цель retention-системы — довести до **7+ касаний в неделю**, каждое из которых даёт
что-то, чего Telegram-канал группы дать не может.

**Принцип:** Telegram получает только тизер — зацепить и направить в приложение.
Полная карточка, детектор, скрытые атрибуты, ачивки, реакции — только внутри.

### 12.2 Недельная карта касаний

| День | Механика | Тип | Заходов |
|---|---|---|---|
| ПН | Старт голосования + прогресс-бар группы | Обязательный | 2 |
| ВТ | Анонимный сигнал «за тебя уже голосовали» | Крючок-интрига | 1–2 |
| СР | Кворумный FOMO — Reveal под угрозой / все ждут тебя | FOMO | 1 |
| ЧТ | Тизер атрибута — эмодзи категории без текста | Предвкушение | 1–2 |
| ПТ | Reveal карточки + детектор + чужие карточки + ачивки | Главный день | 3+ |
| СБ | Реакции на карточки других участников | Социальный | 1–2 |
| ВС | Итоги недели + голосование за вопросы следующего сезона | Подогрев | 1–2 |
| **Итого** | | | **10–14** |

### 12.3 Механики по дням

#### ПН — Старт + прогресс-бар

После завершения своего голосования игрок видит прогресс-бар группы:
«7 из 12 уже проголосовали». Обновляется в реальном времени. Создаёт социальное
давление на не проголосовавших и причину вернуться: *а дотянули ли до кворума?*

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-01 | Прогресс-бар голосования на главном экране группы (X из N проголосовали) | P0 |
| RET-02 | Обновление прогресс-бара в реальном времени (polling каждые 5 мин или WebSocket) | P1 |

#### ВТ — Анонимный сигнал

Пуш во вторник (если пользователь уже проголосовал):
*«Кто-то уже ответил на вопросы про тебя 👀»*

Внутри приложения: только счётчик «X человек уже ответили про тебя» + таймер до Reveal.
Никаких деталей. Чистая интрига — человек возвращается смотреть на число.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-03 | Пуш во вторник 19:00 проголосовавшим участникам | P0 |
| RET-04 | Экран «Ожидание Reveal»: счётчик голосов про тебя + таймер | P0 |
| RET-05 | Счётчик не раскрывает детали — только число | P0 |

#### СР — Кворумный FOMO

Два сценария в зависимости от ситуации в группе:

- Кворум ≥ 80%: пуш не проголосовавшим — *«Все уже ответили, остался только ты»*
- Кворум < 50% в среду вечером: пуш всем — *«Reveal под угрозой, не хватает голосов»*

Никто не хочет быть тем, кто сорвал пятничный Reveal всей группе.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-06 | Пуш в среду 18:00 не проголосовавшим при кворуме ≥ 80% | P0 |
| RET-07 | Пуш в среду 20:00 всем при кворуме < 50%: «Reveal под угрозой» | P0 |
| RET-08 | Снижение порога кворума до 40% для групп < 8 человек | P1 |

#### ЧТ — Тизер атрибута

Самая коварная механика. Пуш в четверг 20:00:
*«Один твой атрибут уже почти определился…»*

Внутри: показываем **только эмодзи категории** (🔥, 🤫, 💪) без текста атрибута.
Человек не знает что именно, весь день думает об этом, в пятницу открывает первым.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-09 | Пуш в четверг 20:00 с тизером | P0 |
| RET-10 | Экран тизера: эмодзи ведущей категории + таймер до Reveal | P0 |
| RET-11 | Алгоритм тизера: категория определяется по лидирующему атрибуту на текущий момент | P1 |

#### ПТ — Reveal как многоступенчатый опыт

Reveal — не один экран. Это последовательность:

1. **Своя карточка** — анимированное открытие, топ-3 атрибута
2. **Ачивки сезона** — новые ачивки с анимацией разблокировки (отдельный экран)
3. **Чужие карточки** — карточки других участников открываются только в приложении,
   в Telegram идёт только тизер-пост
4. **Детектор** — покупка и просмотр списка голосовавших
5. **Скрытые атрибуты** — открытие за кристаллы

Между шагами — естественные паузы и переходы. 20–40 минут в приложении в пятницу вечером.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-12 | Reveal — последовательный flow из 5 экранов, не один | P1 |
| RET-13 | Ачивки сезона — отдельный экран с анимацией после карточки | P1 |
| RET-14 | Карточки других участников видны только внутри приложения | P0 |
| RET-15 | Telegram-пост содержит только тизер, ссылку в приложение, без карточек | P0 |

#### СБ — Реакции на карточки

После Reveal у каждого участника можно поставить реакцию на карточку — 5 фиксированных
эмодзи (😂 🔥 💀 👀 🫡). Не текст, только эмодзи. Получить реакцию на свою
карточку = пуш в субботу = причина зайти.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-16 | Реакции на карточки участников: 5 фиксированных эмодзи | P1 |
| RET-17 | Пуш при получении реакции: «Кто-то отреагировал на твою репу» | P1 |
| RET-18 | Счётчик реакций виден на карточке в профиле группы | P1 |
| RET-19 | Реакции анонимны (кто поставил — не видно) | P0 |

#### ВС — Итоги + выбор вопросов следующего сезона

Два события:

**Летопись группы** — автоматический пост внутри приложения: кто был самым точным
угадывателем, какой вопрос оказался самым спорным (минимальный разброс голосов),
кто получил новую ачивку.

**Голосование за вопросы** — 3 варианта вопроса для следующего сезона, тап на выбор.
30 секунд действия, но заход в приложение есть. Игрок чувствует влияние на игру.

**Требования:**

| ID | Требование | Приоритет |
|---|---|---|
| RET-20 | «Летопись сезона» — автогенерируемый экран с итогами группы за неделю | P1 |
| RET-21 | Пуш в воскресенье 12:00: «Голосуй за вопросы следующей недели» | P1 |
| RET-22 | Экран выбора вопросов: 3 варианта, тап на один | P1 |
| RET-23 | Победивший вопрос гарантированно попадает в ротацию следующего сезона | P1 |

### 12.4 Долгосрочные крючки удержания

#### Стрик голосований

«Ты голосуешь N недель подряд» — отображается в профиле как бейдж и в списке
участников группы. Пропустить сезон = сломать стрик. Работает как Duolingo:
люди возвращаются ради числа, а не ради самой игры.

| ID | Требование | Приоритет |
|---|---|---|
| RET-24 | Счётчик стрика голосований в профиле участника | P1 |
| RET-25 | Бейдж стрика виден в списке участников группы | P2 |
| RET-26 | Пуш в понедельник тем, у кого стрик ≥ 3: «Не прерывай серию» | P1 |
| RET-27 | При пропуске сезона — уведомление «Стрик прерван» с предложением вернуться | P1 |

#### Личный рекорд точности

После каждого Reveal: «Твоя точность угадывания: 67% — личный рекорд!» или
«Ты угадал Машу лучше всех в группе». Повод похвастаться в чате — и органический
шеринг контента про точность.

| ID | Требование | Приоритет |
|---|---|---|
| RET-28 | Экран личной точности после Reveal с историческим сравнением | P1 |
| RET-29 | Выделение «лучший знаток [Имя]» если угадывал конкретного участника точнее всех | P1 |

### 12.5 Стратегия Telegram: воронка, не замена

| Что идёт в Telegram | Что остаётся только в приложении |
|---|---|
| Тизер-пост с намёком на атрибуты | Полная карточка репутации |
| Ссылка на приложение | Скрытые атрибуты 4+ |
| Уведомление о старте сезона | Детектор (кто голосовал) |
| Агрегированные итоги без деталей | Карточки других участников |
| Ачивка сезона группы (без деталей) | Реакции на карточки |
| | Летопись группы |
| | Выбор вопросов следующего сезона |

> Правило: Telegram — афиша. Приложение — сам концерт.

## 11. Риски и митигация

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Буллинг через анонимные голоса | Средняя | Высокое | AI-модерация вопросов, жалобы, ограничение категорий для <18, жёсткий ToS |
| Сторы отклонят внешний биллинг | Низкая | Высокое | Соблюдение гайдлайнов Apple/Google, плашка «внешняя оплата», fallback на IAP |
| Низкий quorum в маленьких группах | Высокая | Среднее | Remind-пуши, снизить quorum до 40% для групп < 8 человек |
| Виральность не сработает | Средняя | Высокое | A/B тест формата карточки, работа с seed-аудиторией (1–2 класса) |
| Утечка анонимности (технически) | Низкая | Критическое | Аудит API, pen-test, изолированные данные голосований |
| Telegram заблокирует бота | Низкая | Среднее | Без бота все функции работают, это enhancement, не core |
| ЮKassa отключит аккаунт | Низкая | Высокое | Резервный провайдер (Тинькофф Касса / CloudPayments) |
| Android фрагментация | Средняя | Среднее | Тестирование на min API 29, Flutter даёт унификацию |
