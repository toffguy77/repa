# T02 — Prisma схема и seed вопросов

## Цель
Создать полную схему БД из master context и засидировать банк вопросов (200+).

## Что реализовать

### prisma/schema.prisma
Реализовать схему точно из master context раздела 4. Все модели, enum'ы, индексы.

Дополнительные индексы для производительности:
```prisma
// Vote
@@index([seasonId])
@@index([targetId, seasonId])

// SeasonResult
@@index([seasonId, targetId])

// Achievement
@@index([userId, groupId])

// UserGroupStat
@@index([userId, groupId])
```

### prisma/seed.ts
Засидировать системные вопросы (source: SYSTEM, groupId: null).

**Минимум 200 вопросов**, распределённых по категориям:

| Категория | Кол-во |
|---|---|
| HOT (горячие) | 40 |
| FUNNY (смешные) | 50 |
| SECRETS (секреты) | 40 |
| SKILLS (способности) | 35 |
| ROMANCE (романтика) | 20 |
| STUDY (учёба) | 20 |

Примеры вопросов для каждой категории (seed должен содержать реальные вопросы, не заглушки):

**HOT:**
- «Кто первым побежит при пожаре?»
- «Кто наябедничает учителю первым?»
- «Кто скорее всего съест чужую еду из холодильника?»
- «Кто притворяется, что всё знает, но на самом деле нет?»
- «Кто тайно читает чужие переписки?»

**FUNNY:**
- «Кто заблудится в трёх соснах?»
- «Кто будет разговаривать с едой перед тем как её съесть?»
- «Кто переименует кота в честь любимого персонажа аниме?»
- «Кто будет смеяться на похоронах?»

**SECRETS:**
- «Кто тайно слушает попсу?»
- «Кто делает вид что не смотрит сериалы, но смотрит?»
- «У кого дома больше всего беспорядка?»

**SKILLS:**
- «Кто выживет последним в зомби-апокалипсис?»
- «Кто лучше всех умеет врать?»
- «Кто первым напишет книгу?»

**ROMANCE:**
- «Кто влюбится первым этим летом?»
- «Кто дольше всех будет переписываться ночью?»

**STUDY:**
- «Кто спишет на контрольной и не попадётся?»
- «Кто будет учить всё в последний день?»
- «Кто получит пятёрку не зная материала?»

Seed должен быть идемпотентным (upsert по тексту вопроса).

### Миграция
Создать первую миграцию: `prisma migrate dev --name init`

## Критерии готовности
- [ ] `npm run db:migrate` выполняется без ошибок
- [ ] `npm run db:seed` загружает 200+ вопросов
- [ ] Все таблицы созданы с правильными индексами
- [ ] Повторный запуск seed не дублирует данные

## Не делать
- Не создавать тестовых пользователей или группы в seed
- Не менять схему относительно master context
