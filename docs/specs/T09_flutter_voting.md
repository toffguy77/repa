# T09 — Flutter: экран голосования

## Цель
Полный UX сессии голосования: вопрос + выбор участника, прогресс, анимации.

## Экраны и компоненты

### VotingScreen (`/groups/:groupId/vote`)
**Структура:**
- Прогресс-бар сверху (X из N вопросов)
- Карточка вопроса с категорией (эмодзи + текст)
- Сетка участников 2×2 (или список если > 4)
- Анимация перехода к следующему вопросу после выбора

**UX детали:**
- Tap на участника → подсветка выбора → через 400ms переход к следующему
- Нельзя выбрать «никого» — обязательный выбор
- Свайп назад заблокирован (нельзя изменить ответ)
- Если голосование прервать и вернуться — прогресс сохранён, пропущенные вопросы не повторяются

### VotingCompleteScreen
После последнего голоса:
- Анимация конфетти (flutter_animate)
- «Ты проголосовал! Reveal в пятницу в 20:00»
- Прогресс группы: «Уже X из N участников проголосовали»
- Кнопка «Назад в группу»

### QuestionCard
- Большой текст вопроса (центрированный)
- Эмодзи категории в углу
- Slide-in анимация при появлении

## Анимации
- Переход между вопросами: slide left
- Выбор участника: scale + border pulse + fade out карточек
- Завершение: конфетти из `flutter_animate` Confetti effect

## Критерии готовности
- [ ] Все вопросы отображаются последовательно
- [ ] Выбор записывается в API
- [ ] Прогресс-бар обновляется
- [ ] После последнего вопроса → VotingCompleteScreen
- [ ] Прерванная сессия продолжается с нужного вопроса
- [ ] Анимации не вызывают jank (60fps)
